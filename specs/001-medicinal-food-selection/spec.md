# Feature Specification: 药食同源电商选品工具

**Feature Branch**: `001-medicinal-food-selection`  
**Created**: 2026-06-23  
**Status**: Draft  
**Input**: User description: "我需要做一个电商选品的工具，包含现在市面上已经正在销售的各种药食同源的产品 ，我可以通过对适用人群\疗效程度、剂型、价格带、利润空间，等等维度进行筛选，能找到这些商品种类的现在代表的品是哪家，销售额是多少，挂网的地址等详细信息"

## Clarifications

### Session 2026-06-23

- Q: 产品分类体系如何定义? → A: 按功能分类为主（免疫力、助眠安神、消化调理、骨骼健康、美容养颜、降三高、补肾益气等），原料类型为辅
- Q: 除5个核心维度外还需要哪些筛选维度? → A: 批准文号/认证类型（蓝帽、SC食品、GMP等）、生产厂家类型（知名药企、专业保健品厂、代工厂等）、销售渠道类型（线上专供、线下专供、全渠道等），另补充产地（道地药材产区）和包装规格维度
- Q: 产品数据如何更新? → A: 仅通过替换JSON数据文件完成更新，v1不需要浏览器内编辑功能
- Q: 产品对比是否需要标注优劣? → A: 需要——在可量化维度上自动标注更优一方（价格更低=优、利润更高=优、销量更高=优），不可量化维度（如剂型、适用人群）并列展示
- Q: v1初始数据规模? → A: 不限定具体数量，但预期数据量级为十万级（10万+）甚至更多，需以此为基准设计数据加载和筛选性能

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 多维度筛选浏览产品 (Priority: P1)

作为一名电商选品人员，我想要通过适用人群、剂型、价格带等多个维度快速
筛选药食同源产品，以便在海量产品中快速定位符合我选品方向的目标商品。

**Why this priority**: 这是工具的核心价值——筛选是用户打开工具后最直接的
需求。没有筛选能力，工具就只是一个静态表格。P1 功能即可独立交付价值。

**Independent Test**: 打开应用 → 选择一个或多个筛选条件（如"中老年人群+
胶囊剂型+中端价格"）→ 查看筛选结果列表。可在无网络环境下完成全流程
测试。

**Acceptance Scenarios**:

1. **Given** 应用已加载全部产品数据，**When** 用户选择一个或多个筛选维度
   （如适用人群=中老年、剂型=胶囊），**Then** 列表实时更新，仅显示同时
   满足所有筛选条件的产品。
2. **Given** 用户已应用筛选条件且存在匹配结果，**When** 用户查看结果列表，
   **Then** 每个产品显示名称、品牌/厂家、剂型、价格、适用人群等关键信息
   摘要。
3. **Given** 用户已应用筛选条件但无匹配结果，**When** 筛选完成，**Then**
   系统显示友好的空结果提示，并建议放宽筛选条件。
4. **Given** 用户未选择任何筛选条件（首次打开），**When** 页面加载完成，
   **Then** 显示全部产品列表，并按默认排序（如销量降序）展示。
5. **Given** 用户已应用筛选条件，**When** 用户清除所有筛选，**Then** 列表
   恢复显示全部产品。

---

### User Story 2 - 查看产品详细信息 (Priority: P2)

作为一名电商选品人员，在筛选出候选产品后，我需要查看某个产品的详细信息
——代表品牌是哪家、参考销售额、挂网地址（电商平台商品链接）等，以便评估
该产品的市场表现和进货可行性。

**Why this priority**: 筛选找到候选品后，必须深入查看详情才能做出选品决策。
P2 建立在 P1 筛选能力之上，是选品决策的关键步骤。

**Independent Test**: 在产品列表中找到任一产品 → 点击查看详情 → 展示包含
厂家、销售额、挂网链接、疗效描述等完整信息。除打开挂网链接外，可在无
网络环境下独立验证所有信息展示。

**Acceptance Scenarios**:

1. **Given** 用户在产品列表页面，**When** 用户点击某个产品，**Then** 展示
   该产品的详细信息面板/页面，包含：产品全称、代表品牌/厂家名称、剂型、
   规格、参考销售额（含数据时间）、适用人群、疗效程度描述、价格区间、
   利润空间预估、电商平台挂网链接列表。
2. **Given** 用户正在查看产品详情，**When** 用户点击挂网链接，**Then**
   在新标签页中打开对应电商平台商品页面（此项操作需要网络连接）。
3. **Given** 用户正在查看产品详情，**When** 用户关闭详情面板或返回列表，
   **Then** 之前应用的筛选条件保持不变。
4. **Given** 某产品有多个挂网链接（不同平台），**When** 用户查看详情，
   **Then** 所有平台链接清晰列出，标注平台名称（如淘宝、京东、拼多多等）。

---

### User Story 3 - 利润空间分析与产品对比 (Priority: P3)

作为一名电商选品人员，我需要对比多个候选产品的利润空间、价格带和疗效
评价，以便在相似产品中做出最优选品决策。

**Why this priority**: 这是高级决策支持功能——在已有筛选和详情查看的基础上，
帮助用户在多个候选品之间做最终决策。对专业选品人员价值很高，但不是初次
使用的基础需求。

**Independent Test**: 勾选2-4个产品 → 点击对比 → 查看并列对比视图，包含
各产品在各维度的对比信息。

**Acceptance Scenarios**:

1. **Given** 用户在产品列表中，**When** 用户勾选2个或更多产品的"对比"
   复选框并点击"开始对比"，**Then** 展示并列对比视图，按维度（价格、
   利润、疗效、剂型、品牌等）逐行对比。可量化维度（价格、利润空间、
   销售额）自动高亮标注更优一方（如：价格更低→绿色标注、利润更高→
   绿色标注），不可量化维度（剂型、适用人群、功能分类等）仅并列展示
   不做优劣判断。
2. **Given** 用户正在查看对比视图，**When** 用户点击某个产品跳转到详情，
   **Then** 返回对比视图后之前的选择状态保持不变。
3. **Given** 用户仅勾选了1个产品，**When** 用户点击"开始对比"，**Then**
   系统提示至少需要选择2个产品才能进行对比。

### Edge Cases

- 产品数据为空（首次部署或数据文件损坏）：显示"暂无产品数据"的友好
  提示，并说明数据更新方式。
- 筛选条件组合产生极端结果（仅1个匹配或0个匹配）：1个结果正常展示，
  0个结果引导用户放宽筛选条件。
- 挂网链接失效或变更：链接标注数据采集日期，用户可理解数据可能存在
  延迟；失效链接不阻止其他信息正常展示。
- 同一产品在多个平台有不同价格：详情中分别展示各平台价格，列表中使用
  最低价作为参考价展示。
- 销售额数据缺失或为估算值：明确标注"估算"或"数据来源说明"，区分精确
  数据与推算数据。
- 移动端浏览器打开：响应式布局确保在手机屏幕上的基本可用性（桌面优先，
  但不应在移动端布局崩溃）。
- 大数据量场景（10万+产品）：数据采用分片加载或 IndexedDB 存储策略，
  首屏仅加载必要数据，筛选和搜索使用索引优化，避免全量遍历导致卡顿。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 以列表/卡片形式展示全部药食同源产品，包含产品
  名称、品牌、剂型、价格等关键摘要信息。
- **FR-002**: 系统 MUST 支持按"适用人群"维度进行多选筛选（如：儿童、
  成人、中老年、孕妇、男性、女性等）。
- **FR-003**: 系统 MUST 支持按"剂型"维度进行多选筛选（如：片剂、胶囊、
  口服液、颗粒、丸剂、膏剂等）。
- **FR-004**: 系统 MUST 支持按"价格带"维度进行筛选（如：50元以下、
  50-100元、100-200元、200元以上）。
- **FR-005**: 系统 MUST 支持按"疗效程度"维度进行筛选（如：保健级、
  调理级、治疗辅助级）。
- **FR-006**: 系统 MUST 支持按"利润空间"维度进行筛选（如：低利润<20%、
  中等利润20-50%、高利润>50%）。
- **FR-007**: 筛选条件 MUST 支持组合使用（多个维度同时生效），筛选结果
  实时更新。
- **FR-008**: 系统 MUST 提供产品详情视图，展示：产品全称、代表品牌/厂家、
  剂型规格、适用人群、疗效描述、参考价格区间、利润空间估算、参考销售额、
  挂网平台链接列表。
- **FR-009**: 每个产品的挂网链接 MUST 标注平台名称和链接采集/验证日期。
- **FR-010**: 系统 MUST 支持关键词搜索功能，可搜索产品名称和品牌/厂家
  名称。
- **FR-011**: 系统 MUST 支持产品对比功能——用户可选择2-4个产品进行逐
  维度并排对比。
- **FR-012**: 所有产品数据 MUST 内嵌于应用中（静态JSON），筛选和搜索
  操作 MUST 在浏览器端完成，不依赖网络请求。
- **FR-013**: 用户的筛选偏好和浏览历史 MUST 保存到浏览器本地存储
  （localStorage），下次打开时恢复状态。
- **FR-014**: 搜索结果和筛选结果 MUST 支持排序（按销量、价格、利润空间
  排序）。
- **FR-015**: 销售额数据 MUST 标注数据类型（精确值/估算值）和数据参考
  期间。
- **FR-016**: 系统 MUST 支持按"功能分类"维度进行多选筛选（如：免疫力、
  助眠安神、消化调理、骨骼健康、美容养颜、降三高、补肾益气等）。
- **FR-017**: 系统 MUST 支持按"批准文号/认证类型"维度进行筛选（如：蓝帽
  标识、SC食品生产许可、GMP认证等）。
- **FR-018**: 系统 MUST 支持按"生产厂家类型"维度进行筛选（如：知名药企、
  专业保健品厂、OEM代工厂等）。
- **FR-019**: 系统 MUST 支持按"销售渠道"维度进行筛选（如：线上专供、
  线下专供、全渠道等）。
- **FR-020**: 系统 MUST 支持按"产地"维度进行筛选（如：宁夏中宁、云南
  文山、吉林长白山、山东东阿等道地药材产区）。
- **FR-021**: 系统 MUST 支持按"包装规格"维度进行筛选（如：单盒装、礼盒
  装、家庭装、散装等），方便评估不同包装形态的选品价值。
- **FR-022**: 产品对比功能 MUST 在可量化维度（价格、利润空间、销售额）上
  自动标注更优一方，不可量化维度（剂型、适用人群、功能分类等）仅做并列
  展示，不做优劣判断。

### Key Entities

- **产品（Product）**: 代表一个药食同源商品。核心属性：名称、品牌/厂家、
  剂型、规格、包装规格、适用人群标签、疗效程度等级、价格区间（最低价、
  最高价）、利润空间百分比、参考销售额（含数据标记）、挂网链接列表、
  产品功能分类（如免疫力、助眠安神、消化调理、骨骼健康、美容养颜、降
  三高、补肾益气等）、产品原料分类（如人参类、枸杞类、灵芝类等）、
  批准文号/认证类型、生产厂家类型、销售渠道类型、产地（道地药材产区）。
- **挂网链接（Listing URL）**: 产品在电商平台的商品页面。属性：平台名称、
  URL地址、平台价格、数据采集日期。
- **筛选条件（Filter Criteria）**: 用户当前选择的筛选维度组合，包含功能
  分类、适用人群、疗效程度、剂型、价格带、利润空间、批准文号、厂家类型、
  销售渠道、产地、包装规格共计11个可选维度，支持多选组合，筛选状态持久
  化到本地存储。
- **对比组（Comparison Set）**: 用户临时选择的2-4个产品集合，用于并列
  对比分析。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可在打开页面后5秒内看到首屏产品列表（首次加载），后续
  筛选操作在1秒内响应（十万级数据集）。
- **SC-002**: 所有筛选条件切换时，结果更新延迟不超过500毫秒（在包含10万+
  产品的数据集上测试）。
- **SC-003**: 用户从打开工具到完成首次筛选并找到目标产品，耗时不超过
  30秒。
- **SC-004**: 关键词搜索在输入停止后300毫秒内返回结果。
- **SC-005**: 产品详情从点击到完整展示所有信息不超过500毫秒。
- **SC-006**: 90%的用户在首次使用时可在不借助帮助文档的情况下完成筛选
  操作。
- **SC-007**: 产品对比功能支持同时对比4个产品时，页面在1秒内渲染完成。
- **SC-008**: 应用在Chrome、Edge、Firefox、Safari主流浏览器的最新版本上
  均可正常使用。
- **SC-009**: 应用首次加载（含核心框架和首屏产品数据）在5秒内完成展示；
  完整产品数据支持分片/懒加载，不影响首屏可用性。

## Assumptions

- 产品数据由运营人员定期整理为JSON格式（可能按品类分片存储以支持
  十万级数据量），作为静态资源随应用一起部署。数据更新通过替换JSON
  数据文件完成，无需后端服务，v1不提供浏览器内数据编辑功能。
- 适用人群分类采用通用分类方式（儿童、成人、中老年、男性、女性等），
  可在产品数据中灵活扩展。
- 疗效程度采用三级划分（保健级、调理级、治疗辅助级），这是基于行业
  惯例的合理推断。
- 价格带和利润空间的数值区间划分可在产品数据配置中调整，不作为硬编码
  逻辑。
- 销售额为参考数据（来自公开渠道或行业报告），非实时精确数据，每条数据
  标注来源和采集时间。
- 挂网链接打开需要网络连接，但产品筛选、搜索和详情查看（除打开外部链接
  外）完全离线可用。
- 目标用户为电商选品从业者，对药食同源产品有基础认知，无需在产品筛选
  工具中进行科普教育。
- 桌面浏览器为主要使用场景，同时保证移动端浏览器的基本可用性（响应式
  布局）。
